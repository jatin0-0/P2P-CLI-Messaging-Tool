Terminal 3 — bob

3
second peer (different port — same machine only)
python tui.py bob alice --port 9001
On separate machines, both peers use the default port — no --port needed. The flag is only required when sharing one machine.

Connect to multiple peers at once

alice talks to bob and carol simultaneously
python tui.py alice bob carol

Add a peer while running

Clone

git clone https://github.com/jatin0-0/P2P-CLI-Messaging-Tool.git
cd P2P-CLI-Messaging-Tool

Create a virtual environment (recommended)

python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

Install dependencies

pip install textual aiosqlite aiohttp flask plyer
plyer is optional — OS popup notifications work if installed, silently skipped if not.
flask is only needed on the machine running the rendezvous server.

Verify

python -c "import textual, aiosqlite, aiohttp; print('all good')"

File naming — important on Windows

Windows is case-insensitive but Python imports are not. Make sure filenames are lowercase:

peer_manager.py   ✓
PeerManager.py    ✗  (causes circular import)
# rename in PowerShell if needed
Rename-Item PeerManager.py peer_manager.py
Rename-Item PeerWorker.py  peer_worker.py

Cross-network (different Wi-Fi / mobile hotspot)

Deploy rendezvous_server.py on any VPS, then edit config.py:

RENDEZVOUS_HOST = "YOUR_SERVER_PUBLIC_IP"

Both peers run tui.py normally. The tool auto-picks the best path.

Connection paths (automatic, no flags)

Path A — direct LAN
Same /24 subnet (e.g. both on 192.168.1.x)
Path B — UDP hole punch
Different networks, cone NAT — 3 s timeout
Path C — HTTP relay
Symmetric NAT or no public IP — routes via rendezvous server

History

All messages are stored in chat.db (SQLite). History loads automatically when you open a peer tab — no setup needed.

One UDP socket, bound once. All peers share it. Packets routed by source address.

Only tui.py and rendezvous_server.py are entry points. Never run the others directly.

tui.pyEntry point — Textual UI, tabs, input bar, notifications
peer_manager.pyOwns the UDP socket, routes packets to workers
peer_worker.pyOne asyncio task per peer — full connect/chat/reconnect state machine
storage.pySQLite via aiosqlite — messages + peer address records
stun_client.pyDiscovers public IP/port via STUN (RFC 5389) — unchanged
rendezvous_server.pyFlask meeting-point server — unchanged from original
config.pyAll tunable constants including DB_PATH
chat.dbCreated automatically on first run — do not commit to git
